# SQLZoo Practice Repository


## Overview
This repository documents my full walkthrough of the SQLZoo tutorials and quizzes. Each section includes the original problem, my solution, and an in‑depth explanation of the reasoning behind the query. The goal of this project is to demonstrate not only correct SQL syntax, but also clear thinking, relational modeling, and problem‑solving discipline.
SQLZoo covers a wide range of SQL concepts, including:
• 	basic SELECT statements
• 	filtering and sorting
• 	joins and subqueries
• 	grouping and aggregation
• 	self‑joins and multi‑table relationships
• 	path‑finding logic using relational data
This repo serves as both a study reference and a portfolio artifact showing how I break down problems and explain my approach.

## Structure
Each folder corresponds to a SQLZoo section. Inside each folder, every problem includes:
• 	Problem Summary — a short description of what the question is asking
• 	Explanation — a step‑by‑step breakdown of the logic behind the query
• 	Final Query — the SQL solution I wrote
This format keeps the repo consistent and makes it easy to revisit concepts later.

## Why This Project Matters
SQLZoo is more than a set of exercises. It’s a progression of relational thinking:
• 	early problems build comfort with SELECT and WHERE
• 	mid‑level problems introduce multi‑table joins
• 	advanced problems require chaining relationships, interpreting schemas, and modeling real‑world logic
• 	the final self‑join problems (especially Problems 9 and 10) require graph‑style reasoning
Documenting each step shows how I approach unfamiliar schemas, break down complex joins, and translate real‑world scenarios into SQL.

## Key Learning Highlights
• 	Join discipline — understanding when to join on keys, when to join on shared attributes, and how to avoid accidental Cartesian products
• 	Alias clarity — using meaningful aliases to track multiple roles of the same table
• 	Schema interpretation — reading table structures and inferring relationships
• 	Self‑joins — modeling multi‑step paths using repeated references to the same table
• 	Query readability — writing SQL that is clean, consistent, and easy to follow
• 	Problem decomposition — breaking complex tasks into sequential, logical steps
These skills translate directly into backend development, data analysis, and any environment where relational data is used.

## Final Notes
This repository represents a full, disciplined pass through SQLZoo, including the advanced problems that require multi‑stage reasoning. It’s both a learning archive and a demonstration of my ability to explain technical decisions clearly.
